# v0

**What it is:** Vercel's prompt-to-app builder, deliberately constrained to a single output stack (Next.js + Tailwind + shadcn/ui) and running on a real remote sandbox rather than in-browser.
**Axis:** app-builder, deploy.
**Depth:** medium — sandbox and shadcn-registry mechanics are well-documented; token/eval figures not sought.

## Products & surfaces

- **v0** — chat-to-app, emits standard React/Next.js + Tailwind + shadcn/ui, backed by a Git panel (branch → diff → PR).
- **Vercel Sandbox** (internal codename **Hive**) — the execution substrate: Firecracker microVMs, in production for Vercel's own builds since late 2023, GA'd as a standalone product; also used by Blackbox AI and RooCode.
- **shadcn/ui registry** — a distribution spec, not a component library, that v0 (and other tools) can consume.

## Notable features

| Feature | What it does | Steal? |
|---|---|---|
| Vercel Sandbox / Hive | Firecracker microVMs, dedicated kernel per sandbox, ~125ms boot, isolated from prod env vars/DB connections/cloud resources | yes |
| Git panel (branch → diff → PR) | Every prompt lands as a reviewable, mergeable artifact in a real repo | yes |
| GitHub import + auto env-var pull | Import any repo, Vercel auto-pulls matching env vars | yes |
| shadcn registry as machine-readable design system | `Open in v0` prepopulates a chat from a registry endpoint (`/r/<component>.json`) supplying components, theme tokens, and metadata | yes |
| Constrained output stack | v0 only emits Next.js/Tailwind/shadcn, not an arbitrary framework | yes, as a design principle |

`Steal?` is a first-pass signal only, not a decision.

## Worth stealing

### shadcn as the strategic escape hatch

shadcn/ui's foundational move is that it **copies component source into your repo** rather than shipping an importable npm library — "the markup, Tailwind classes, and CVA variants belong to your repo." That single decision is what makes v0's output *yours* in a way that's structurally different from Base44 or (partially) Lovable: there's no proprietary runtime dependency to strip out later, because there never was one.

### The registry as a distribution spec for models, not humans

A shadcn registry item is JSON: component source, dependencies, theme variables, metadata. `Open in v0` is literally a URL — `v0.dev/chat/api/open?url=<registry-item-url>` — that prepopulates a chat pointed at that endpoint. shadcn's own docs frame this directly: *"A registry is a distribution specification designed to pass context from your design system to AI Models."* This reframes a design system as a **machine-distributable artifact for a model to consume**, not a Figma file for a human to reference. The registry, not the chat prompt, is the unit of design-system distribution.

### Constraint as the thing that makes generation coherent

v0's output space is deliberately narrow — Next.js, Tailwind, shadcn, nothing else. That's the mechanism, not an accident: a smaller, well-trodden target space is what makes multi-file, multi-component generation hang together instead of drifting into inconsistent patterns. The lesson generalizes past v0 — an app-builder that supports "any stack" trades coherence for breadth.

### Git panel and PR as the reviewable artifact

Same shape as Ramp's Inspect (see `ramp.md`): the agent's output lands in a medium — a diffable, mergeable PR — that already has review, history, and rollback built in, rather than a bespoke "agent changes" panel.

## Worth avoiding

Nothing clearly identified from this pass — v0's public material is mostly feature description, not incident postmortems. Absence of a public security-incident record is not evidence of absence; just not surfaced here.

## Facts & figures

- Firecracker microVMs: ~125ms boot time, few MB memory footprint each (Firecracker's own general figures, applied to Hive by Vercel/InfoQ coverage).
- Hive has powered Vercel's own build infrastructure since late 2023; Vercel Sandbox (the productized version) reached GA afterward.
- `Open in v0` explicitly does **not** support `cssVars`, `css`, `envVars`, namespaced registries, or advanced auth — a documented limitation, not an oversight.
